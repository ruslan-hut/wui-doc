# GetSetingsRRO

Повертає XML налаштувань РРО (касового апарата), включно з атрибутом
`license` пристрою. Ми розбираємо ліцензію, щоб керувати значком у списку
пристроїв (активна / прострочена / пробна / безкоштовна тощо).

Зверніть увагу на друкарську помилку в назві COM-методу: `GetSetingsRRO`
(одна `t`). Збігайте її точно під час виклику.

Двокрокова схема. Білдер: `olexml.DeviceXML` (лише `FN`).

## Вхід

```xml
<InputParameters><Parameters FN="4000167785" /></InputParameters>
```

## Вихід

Повертає великий документ налаштувань РРО. Атрибут ліцензії — це те, що нас
цікавить:

```xml
<OutputParameters>
  <Parameters ... license='29.10.2029' ... />
</OutputParameters>
```

Формати `license`:

| Формат               | Приклад         | Значення |
|----------------------|-----------------|---------|
| `DD.MM.YYYY`         | `29.10.2029`    | Спливає в цю дату |
| іменований (не дата) | `free` / `demo` | Безстрокова ліцензія цього типу |

Парсер: `parseLicenseFromRROXML` → `parseLicenseDate` у
`internal/service/ole_service.go`. Обчислює статус, кількість днів, що
залишилися, та дату закінчення для UI.

## Викликач

`OLEService.LoadLicensesAsync` (викликається під час запуску) та
`OLEService.ReloadLicenses` (після перезавантаження). Спочатку пробує HTTP
endpoint ліцензії, якщо налаштовано (`LicenseHTTPClient`), повертається до
OLE як резервний варіант. Результат транслюється всім WS-клієнтам як
`device.license.updated`.

# AddOperator

Реєструє або оновлює оператора в базі даних пристрою. Використовується після
збереження нового ключа (схема `SaveKey`), щоб драйвер WebCheck міг
використовувати цей ключ для підписання чеків.

Двокрокова схема. Білдер: `olexml.AddOperatorXML`.

## Вхід

```xml
<InputParameters>
  <Parameters fn="4000167785" operatorname="..." keypath="C:\\Path\\to\\key.dat"
              keypass="..." inn="1234567890"/>
</InputParameters>
```

| Атрибут         | Значення |
|-----------------|---------|
| `fn`            | Фіскальний номер пристрою |
| `operatorname`  | Відображуване ім'я оператора (CN із сертифіката) |
| `keypath`       | Абсолютний шлях до файлу `.key.dat` |
| `keypass`       | Пароль ключа |
| `inn`           | ІПН оператора (ДРФО/ЄДРПОУ) |

> ⚠️ Назви атрибутів **у нижньому регістрі** лише для цього методу. Усі
> інші методи використовують `FN` (верхній регістр). Дотримуйтесь того, що
> очікує драйвер для кожного методу — див. [особливості](quirks.md).

> Не-ASCII символи в `operatorname` (наприклад, кирилиця) кодуються як
> числові посилання на символи XML (`&#x041A;`) функцією `escapeXMLAttr`,
> тож корисне навантаження є чистим ASCII під час запису у stdin
> VBS-воркера. Це уникає пошкодження кодової сторінки stdin.

## Вихід

Успіх:

```xml
<OutputParameters>
  <Parameters Err="0" AddNewOperator="NAME" INN="1234567890" UpDate="1"/>
</OutputParameters>
```

Помилка:

```xml
<OutputParameters>
  <Parameters Err="1004" ErrHelp="Неверный формат XML." version="6.0.7.1338"/>
</OutputParameters>
```

| Атрибут          | Значення |
|------------------|---------|
| `Err`            | `"0"` = успіх; ненульове значення = код помилки |
| `AddNewOperator` | Відлуння імені оператора |
| `INN`            | Відлуння ІПН |
| `UpDate`         | `"1"` = оновлено наявного, `"0"` = додано нового |
| `ErrHelp`        | Опис помилки кирилицею |

Парсер: `parseAddOperatorStatusXML`. Результат: `dto.AddOperatorResult` з
полями `Updated` / `Name` / `INN` / `Error`.

## Викликач

`OLEService.AddOperator` → `OLEKeyAdapter` (використовується `KeyHandler`
після того, як `SaveKey` записує файл ключа). Повторює спробу один раз у разі
помилки каналу, якщо воркер перезапускається посеред виклику.

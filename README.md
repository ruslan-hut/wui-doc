# WUI — Веб-панель керування для WebCheck POS

Сучасна веб-панель для керування програмними реєстраторами розрахункових операцій (пРРО) [WebCheck](https://www.webchek.com.ua/ru/programmniy_rro/). Моніторинг POS-терміналів у реальному часі, перегляд історії транзакцій, формування фіскальних звітів та керування налаштуваннями пристроїв — усе з браузера.

> **Дзеркало документації.** Цей репозиторій є відкритим дзеркалом документації приватного проєкту WUI, призначеним для спільного використання конфігурації та опису проєкту між супровідниками (maintainers). Уся документація перекладена українською. Програмний код, ключі конфігурації, назви типів повідомлень, SQL та інші технічні ідентифікатори збережені мовою оригіналу.

---

## Що таке WUI?

WUI — це додаток-компаньйон до [WebCheck](https://www.webchek.com.ua/ru/programmniy_rro/), програмного фіскального реєстратора (пРРО), який використовується в українській роздрібній торгівлі. Він надає зручний браузерний інтерфейс для всього, що зазвичай виконується через десктопний застосунок WebCheck, але швидше та наочніше.

WUI працює як легкий локальний сервіс на тій самій машині Windows, де встановлено WebCheck. Він безпосередньо читає бази даних та конфігураційні файли пристроїв, відображає дані у вигляді чистої панелі керування та передає команди назад до WebCheck через OLE-автоматизацію. Інтернет не потрібен — усе працює локально.

---

## Архітектура

WUI — це єдиний виконуваний файл (`wui.exe`), який поєднує Go-бекенд з веб-інтерфейсом на Angular, скомпільованим у бінарний файл. Уся комунікація між браузером і бекендом відбувається через єдине з'єднання WebSocket — без REST API, без опитування (polling).

**Бекенд**: Go 1.22+, маршрутизатор `chi`, `gorilla/websocket`, конфігурація `cleanenv`, логування `slog`, БД `sqlite`, `embed.FS` для UI
**Фронтенд**: Angular (standalone-компоненти), Material Design 3, SCSS + CSS custom properties, світла/темна теми
**Комунікація**: лише WebSocket (JSON, кореляція запит-відповідь через `requestId`)

## Можливості

- **Моніторинг пристроїв** — стан фіскальних реєстраторів у реальному часі, сповіщення про офлайн-статус та закінчення ліцензій
- **Історія змін та чеків** — перегляд фіскальних змін і окремих чеків з детальним попереднім переглядом
- **Фіскальні звіти** — формування X-звітів, Z-звітів та періодичних звітів безпосередньо з панелі
- **Керування ліцензіями** — моніторинг термінів дії та замовлення нових ліцензій
- **Оператори та форми оплати** — перегляд і редагування списку операторів та доступних форм оплати
- **Налаштування пристроїв** — редагування конфігурації через структуровану форму, перегляд логів
- **Українська локалізація** — повний кириличний інтерфейс з автоматичним перетворенням кодування Windows-1251 → UTF-8
- **Світла та темна теми** — з визначенням системних налаштувань

## Обробка дати/часу (ВАЖЛИВО)

SQLite зберігає дати як рядки **локального часу** (`YYYY-MM-DD HH:MM:SS`, без часового поясу). Використання `new Date(string)` спричиняє зсув **+2 години** (Україна, EET/EEST). Правило: ніколи не використовувати `new Date(string)` або `time.Time` для цих дат — парсити компоненти безпосередньо.

## Кодування тексту: Windows-1251 / UTF-8 (ВАЖЛИВО)

POS-застосунок (WebCheck) записує файли у кодуванні **Windows-1251**. Go/JSON вимагають UTF-8. Виявлення — через `utf8.Valid()`: якщо валідне — використовувати як є, якщо ні — декодувати через `charmap.Windows1251.NewDecoder()`.
